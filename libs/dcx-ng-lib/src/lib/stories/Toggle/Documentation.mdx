# Toggle

The `dcx-ng-toggle` is a flexible and accessible switch component used to represent on/off states.  
It is designed for easy customization and can be styled or left unstyled based on project needs.

---

## Unstyled Version

Use this version when you want full control over the visual appearance.  
No default styles are applied — only the structure and behavior remain.

## Class-Based Version

This version includes default styling using BEM-based classes and minimal SCSS.  
It supports customization through size input. The active color follows the design tokens and is not configurable via component properties.

### Available Properties

- **checked**: Whether the toggle is in the "on" state
- **disabled**: Disables the toggle interaction if `true`
- **label**: Optional label text shown next to the toggle
- **ariaLabel**: Used for accessibility when no visible label is provided
- **size**: `'small' | 'medium' | 'large'` — controls the size of the toggle
- **color**: Any valid CSS color — controls the active background when checked
- **(toggled)**: Event emitted with the new `checked` value whenever the toggle is clicked or toggled with keyboard

## Manual Usage Example

The toggle works with basic property binding and event handling:

```html
file.HTML
<dcx-ng-toggle
  [checked]="featureEnabled"
  label="Enable Feature"
  size="medium"
  
  (toggled)="handleToggle($event)"
></dcx-ng-toggle>


file.TS
handleToggle(value: boolean) {
    this.featureEnabled = value;
  }
```
